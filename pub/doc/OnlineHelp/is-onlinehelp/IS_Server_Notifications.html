Notifications
Notifications provide visibility into events inside Integration Server and help administrators observe system behavior, identify issues, and take appropriate actions to minimize any service degradation or disruption.
Notifications are displayed in Integration Server Administrator when system-defined or user-defined criteria are met. For example, a notification about a user's password expiry is generated depending on the value of the Expiration Email Remainders field on the Security > User management > Password expiration settings page. The default value of Expiration Email Remainders is 3, which indicates that a notification about the user's password expiry is displayed three days before the password expires.
Each notification has an icon that represents the severity of the event, a subject, time of the notification, notification type, and a description. A description with multiple lines of text is displayed as a single line ending with ellipses, which indicates that the remaining lines are displayed on hover. If there are multiple unread notifications, scroll to the required notification to read it.

Dismiss this notification
Marks a notification as read. Dismissing a notification marks the notification as read and clears it for all the users who might log on to Integration Server Administrator. Also, you cannot retrieve a dismissed notification from Integration Server Administrator.
Ignore such notifications
Stops displaying notifications of the selected type.
Additional actions
Dismiss or mark all notifications as read or view ignored notification types.
Dismiss all
Dismisses or marks all notifications as read.
Ignored notifications
View all the notification types and their status. (Ignore or Report). For a notification type, Ignore indicates that user does not receive notifications of that type and Report indicates that user continues to receive notifications of that type.
Report all notification types
Subscribes to all notification types.
