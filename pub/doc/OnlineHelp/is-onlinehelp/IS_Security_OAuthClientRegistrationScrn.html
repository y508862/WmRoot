Client Registration
Use the Client Registration page to view all clients registered with the authorization server.
Click Return to OAuth to return to the OAuth.
Click Register Client to register a new client.
Registered Clients
Client Application
The name and version number of the client in name (version) format.
Client ID
The client ID Integration Server generated for the client when the client was registered.
Client Type
Specifies the type of the client according to its ability to communicate with the authorization server as follows:
Option
Description
Confidential
Indicates that the client can maintain the confidentiality of its credentials or is capable of secure client authentications.
Public
Indicates that the client is not capable of maintaining confidentiality of its credentials and not capable of secure client authentications.
Active
Displays the status of the client. When the client is enabled, the authorization server can accept requests from that client. You can block incoming requests from a client by disabling it. To disable a client, click the Yes link in this column. To re-enable a client, click the No link in this column.
When a client account is disabled and watt.server.oauth.disableClient.disableTokens is true, the authorization server will not accept requests for authorization from the client. Furthermore, when watt.server.oauth.disableClient.disableTokens is true, all tokens issued to that client account will be considered inactive. The introspection endpoint (pub.oauth:introspectToken) returns "active=false" for all tokens issued to a disabled client account. When a client account is re-enabled all tokens issued to that client that have not expired or been revoked are re-activated.
Delete
Deletes the client specified in the Client Application column. To delete a client, click the icon in the Delete column.
