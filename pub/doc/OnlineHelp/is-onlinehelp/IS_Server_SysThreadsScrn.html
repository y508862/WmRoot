System threads
Use the System threads page to view the status of the server’s system threads.
Click Return to Server Statisticsto return to the main statistics page.
Check Show threads that can be canceled or killed at the top of the list to immediately change the display to show threads that can be killed or canceled, at the top of the list.
System Threads
Priority
Displays the order in which the threads are run. Threads with a higher priority run before those with lower priority. For example, threads with a priority of 5 run before threads with a priority of 4. The threads with the same priority level can be scheduled to run one at a time or to run concurrently, depending on what the Java Virtual Machine can handle.
Name
Displays the name of the thread.
Alive
Indicates whether or not the thread has been started and has not died. A green check mark indicates that the thread has been started.
Daemon
Indicates whether or not the thread is a daemon (services other threads). A green check mark indicates that the thread is a daemon.
Interrupted
Indicates whether or not the thread has received an interrupt signal, which may indicate that an error has occurred. If your server is not behaving as expected, the status of this field can indicate whether an internal process has malfunctioned.
Cancel
Indicates whether the thread can be canceled. If a yellow check mark is present, you can click it to cancel the thread. When you cancel a thread, Integration Server frees up resources that are held by the thread and returns the thread to a threadpool from which it originated. For more information about canceling and killing threads, refer to webMethods Integration Server Administrator’s Guide.
Kill
Indicates whether the thread can be killed. If a red X () is present, you can click it to kill the thread. You must first try to cancel a thread before Integration Server will allow you to kill a thread. When you kill a thread, Integration Server terminates the thread, possibly leaving associated resources in an unknown state. Integration Server then adds a new thread to the associated threadpool. For more information about canceling and killing threads, refer to webMethods Integration Server Administrator’s Guide.
Start Time
Indicates the time when the task started.
Protocol
Indicates the transmission protocol (for example, HTTP or FTP) the thread is using.
