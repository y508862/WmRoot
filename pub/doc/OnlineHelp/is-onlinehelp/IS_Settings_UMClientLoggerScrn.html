View UM client logger details
Use this page to view settings for the UM Client logger which writes to the Universal Messaging client log file named umClient.log. This page also displays the configuration for event lifecycle logging
Click Edit UM Client Logger to change the settings for the session logger.
UM Client Logger Configuration
Log Level
Level of the information that is written to the log. Each level outputs log entries with that level or higher.
Log Size
Maximum size of the log file. When this size is reached, Integration Server rolls the file over to a backup called umClient.log.number and creates a new file.
Log Depth
Number of backup log files to keep on disk when using log rolling. When this number is reached, Integration Server deletes the oldest log file.
UM Client Event Lifecycle Log Configuration
Trace Store Log Level
Logging level for the trace logger. Event lifecycle logging, also called trace logging, uses a trace logger to create detailed logs describing the flow of a message (event) from the publishing UM client through the Universal Messaging server to the subscribing UM client.
Log Level
Description
Off
Event lifecycle logging is disabled for this Integration Server. This is the default value.
Info
Event lifecycle logging is enabled. A trace logger writes info level messages for the queues or channels specified in Trace Stores.
Trace
Event lifecycle logging is enabled. A trace logger writes trace level messages for the queues or channels specified in Trace Stores. The Trace log level is more detailed than the Info log level.
Trace Stores
Comma-separated list of the message stores, specifically queue names or channel names, for which event lifecycle logging is performed. Specify * for trace loggers to be used with all queues or channels that this Integration Server sends messages to and receives messages from. The default is *.
Trace Folder Log Size
The maximum size in MB of all the log files for event lifecycle logging. That is, the maximum size of all the log files in the Integration Server_directory /instances/instanceName/logs/umEventLifecycle. The default is 1024.
Trace Store Log Size
The maximum size in MB of the trace log file for an individual message store. That is, the maximum size of an individual log file in Integration Server_directory /instances/instanceName/logs/umEventLifecycle. The default is 10.
