Store settings
The Store settings page identifies the location and initial size for the document stores and the XA recovery store used by the Integration Server. Integration Server maintains the following stores:
The default document store contains default documents delivered to the default Broker client for the Integration Server. When Integration Server retrieves documents delivered to its default Broker client, Integration Server places the documents in the default document store. Documents remain in the default document store until the dispatcher determines which webMethods messaging triggers subscribe to the document. The dispatcher then moves the documents to the trigger queues for the subscribing triggers.
The trigger document store contains locally published guaranteed documents that are delivered to specific webMethods messaging triggers. Integration Server creates a queue for each trigger in the trigger document store. A document remains in the trigger document store until the server successfully executes the service specified in the trigger or the server discards the document because it does not satisfy any of the trigger conditions.
The outbound document store, sometimes called the client-side queue, contains published documents waiting to be sent to the Broker. Integration Server places documents in the outbound document store when the configured Broker is disconnected. Integration Server sends documents from the outbound document store to the Broker when the connection to the Broker is restored.
The XA recovery store saves the XID and the global state of XA transactions. At the beginning of each subsequent action taken for the transaction, Integration Server stores the global state of the transaction and the state of each resource that is participating in the transaction in the XA recovery store. If Integration Server shuts down unexpectedly, Integration Server can retrieve the state information from the XA recovery store and try to resolve uncompleted transactions, if there are any.
Click Edit Document Store Settings to edit the settings for the default, trigger, or outbound document stores.
Click Edit XA Recovery Store Settings to configure the XA recovery store.
Default Document Store
Store Location
The location of the default document store. By default, the Integration Server saves document stores in the following directory:
Integration Server_directory\instances\instance_name\DocumentStore
Initial Store Size (MB)
The amount of disk space allocated to the default document store at start up. The default store automatically increases when it receives data that exceeds the initial size. The default size is 25 MB.
Capacity (documents)
The maximum number of documents in the default document store. The default is 10 documents.
Note:
The Capacity field displays "Suspended" next to the field if the Capacity was set to 0 (zero) on the Edit Document Store Settings screen. The Capacity field displays "Broker Not Configured" if there is not a Broker configured for the server.
Refill level (documents)
The number of unprocessed documents that remain in the default document store before Integration Server retrieves more documents for the store from the Broker.
Note:
The Refill level field displays "Broker Not Configured" if there is not a Broker configured for the server.
Note:
The difference between the Capacity and the Refill level determines how many documents Integration Server retrieves for the default document store from the Broker. For example, if you assign the trigger document store a Capacity of 10 and a Refill level of 4, Integration Server initially retrieves 10 documents for the default store. When only 4 documents remain to be processed in the default store, Integration Server retrieves 6 more documents. If 6 documents are not available, Integration Server retrieves as many as possible.
Trigger Document Store
Store Location
The location of the trigger document store. By default, Integration Server saves document stores in the following directory:
Integration Server_directory\instances\instance_name\DocumentStore
Initial Store Size (MB)
The amount of disk space allocated to the trigger document store at start up. The trigger document store automatically increases when it receives data that exceeds the initial size. The default size is 35 MB.
Inbound Document History (minutes)
The number of minutes the server maintains a history for a received document. During this time period, the server will perform duplicate detection for the document. If the server receives another document with the same document ID, the server considers the second document to be a duplicate and discards it. A value of 0 (zero) indicates that the server does not check for duplicate messages. The default is 10 minutes. This field displays "Not Available" if Integration Server connects to a Broker version 6.1 or later.
Note:
The Inbound Document History (minutes) field can be set only if Integration Server connects to a Broker version 6.0.1. If Integration Server connects to a Broker version 6.1 or later, you can configure duplicate detection on a per trigger basis. For information about configuring duplicate detection for triggers using version 6.1 or later of Integration Server, see the Publish-Subscribe Developer’s Guide.
Inbound Client Side Queuing
Indicates whether client side queuing is On or Off.
If you enable client-side queuing (On), Integration Server maintains persistent trigger document stores on disk. If the server shuts down unexpectedly before processing the document, the server recovers the document from the document store upon restart.
If you disable client-side queuing (Off), Integration Server maintains the trigger document store in memory and the Broker maintains the persistent queue. If the server shuts down unexpectedly before processing the document, the server recovers the document from the Broker.
If Integration Server connects to a 6.1 or later version of the Broker, this field displays "Not Available".
Note:
Client-side queuing is only available if Integration Server connects to a 6.0.1 version of the Broker.
Run Trigger Service As User
Specifies the user account whose credentials the server uses to execute a service specified in a trigger condition for a webMethods messaging trigger that receive documents from the Broker. This field can specify a predefined user account or a user account created by a server administrator. The default user is Administrator.
Administrator. Used to access the Integration Server Administrator to configure and manage the server.
Default. Used when the client does not supply a user name and password.
Developer. Used to connect to the server from Software AG Designer to create, modify, and delete services that reside on the server.
Replicator. Used during package replication.
Note:
This setting applies to webMethods messaging triggers only.
Outbound Document Store
Maximum Documents to Send per Transaction
The number of documents the server should send from the outbound document store to the Broker for each transaction. The default is 25 documents.
Note:
The Maximum Documents to Send per Transaction field displays "Broker Not Configured" if a Broker is not configured for the server.
XA Recovery Store
Store Location
The location of the XA recovery store. By default, the Integration Server saves the XA recovery store in the following directory:
Integration Server_directory \instances\instance_name\XAStore
Initial Store Size (MB)
The amount of disk space allocated to the XA recovery store at start up. The XA recovery store automatically increases when it receives data that exceeds the initial size. The default size is 10 MB.
