XA manual recovery
Use the XA manual recovery page to view and resolve uncompleted XA transactions that the Integration Server cannot resolve automatically.
Note:
Refresh the XA manual recovery page at intervals to make sure all uncompleted transactions are listed. Suppose Integration Server tries to resolve an uncompleted transaction but cannot; the transaction will not be listed while Integration Server is trying to resolve it, but if you refresh the screen later, the transaction will appear on the list.
Click Return to Resource Settings to return to the list of current resource settings.
Click Disable XA Transaction Recovery to disable XA transaction recovery. When you disable XA transaction recovery, Integration Server does not maintain XID or state information for XA transactions. Integration Server cannot be used to recover incomplete transactions.
Click Enable XA Transaction Recovery to enable XA transaction recovery. When you enable XA transaction recovery, Integration Server maintains XA transaction information (such as XID, global transaction state, and the state of each participating resource) in the XA recovery store. You can use Integration Server to recover transactions.
Unresolved XA Transactions
Displays all of the uncompleted XA transactions that Integration Server could not resolve automatically. If you want to resolve these transactions, you will need to do so manually.
Note:
The Unresolved XA Transactions table does not appear if you disabled XA transaction recovery.
XID
Unique XID for the transaction. Integration Server creates the XID at the beginning of the transaction and writes it to the XA recovery store; Integration Server also provides the XID to the participating resources, which also store the information.
Click the XID to view detailed information about the transaction, such as the resources involved, the state of the transaction on each resource, and the inferred status of the transaction.
Global 2PC State
Global state of the transaction before Integration Server ended.
Error Message
Entry that Integration Server wrote to its server log before storing the global state of the transaction in the XA recovery store.
Recovery Action Attempted
Action that Integration Server took to try to resolve the transaction. If Global 2PC State is TR_COMMIT_BEGIN, Integration Server tried to commit the transaction. If the global state is any other value, Integration Server tried to roll back the transaction.
