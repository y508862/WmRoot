Edit document store settings
The Integration Server uses document stores to save published documents to disk or to memory while the documents wait for processing. Document stores ensure that once a guaranteed document is persisted, it cannot be lost. The Integration Server maintains three document stores for published documents.
The default document store contains default documents delivered to the default Broker client for the Integration Server. When Integration Server retrieves documents delivered to its default Broker client, Integration Server places the documents in the default document store. Documents remain in the default document store until the dispatcher determines which webMethods messaging triggers subscribe to the document. The dispatcher then moves the documents to the trigger queues for the subscribing triggers.
The trigger document store contains locally published guaranteed documents that are delivered to specific webMethods messaging triggers. Integration Server creates a queue for each trigger in the trigger document store. A document remains in the trigger document store until the server successfully executes the service specified in the trigger or the server discards the document because it does not satisfy any of the trigger conditions.
The outbound document store, sometimes called the client-side queue, contains published documents waiting to be sent to the Broker. Integration Server places documents in the outbound document store when the configured Broker is disconnected. Integration Server sends documents from the outbound document store to the Broker when the connection to the Broker is restored.
Note:
An asterisk (*) next to a field indicates that you need to restart the server for changes to take effect.

Default Document Store
Store Location
Specify the location of the default document store. By default, the Integration Server saves document stores in the following directory:
Integration Server_directory\instances\instance_name\DocumentStore
If you want to save the default document store in a different location, specify the directory in this field. If the directory does not exist, the server creates it.
Important:
Make sure that you have write access to the specified directory and that the directory path does not contain any characters considered illegal by your operating system.
Initial Store Size (MB)
Specify the amount of disk space to allocate to the default document store at start up. The default store automatically increases when it receives data that exceeds the initial size. The default size is 25 MB.
When setting the Initial Store Size, consider the size and volume of documents that you expect to be delivered to the default document store. If you expect large documents or a high volume of documents, consider increasing the Initial Store Size. If you expect small documents or a low volume of documents, you might want to use the default size.
Important:
Make sure that there is enough available disk space on the Integration Server machine to accommodate the initial sizes of the default document store and the trigger document store.
Capacity (documents)
Specify the maximum number of documents in the default document store. The Capacity must be greater than the Refill Level. The default is 10 documents.
Note:
If you set Capacity to 0 (zero), the server automatically sets the Refill Level to 0 (zero). If you set the Capacity field to 0 (zero), the server displays "Suspended" next to the field on the Store Settings screen.
Note:
The Capacity field displays "Broker Not Configured" if there is not a Broker configured for the server.
Refill Level (documents)
Specify the number of unprocessed documents that remain in the default document store before the Integration Server retrieves more documents for the store from the Broker. The Refill Level must be less than Capacity. The default is 4 documents.
Note:
The Refill Level field displays "Broker Not Configured" if there is not a Broker configured for the server.
 
Trigger Document Store
Store Location
Specify the location of the trigger document store. By default, the Integration Server saves document stores in the following directory:
Integration Server_directory\instances\instance_name\DocumentStore
If you want to save the trigger document store in a different location, specify the directory in this field. If the directory does not exist, the server creates it.
Important:
Make sure that you have write access to the specified directory and that the directory path does not contain any characters considered illegal by your operating system.
Initial Store Size (MB)
Specify the amount of disk space to allocate to the trigger document store at start up. The trigger document store automatically increases when it receives data that exceeds the initial size. The default size is 35 MB.
Note:
Make sure that there is enough available disk space on the Integration Server machine to accommodate the initial sizes of the default document store and the trigger document store.
Inbound Document History (minutes)
Specify the number of minutes the server maintains a history for a received document. Specify a value of 0 (zero) if you do not want the Integration Server to check for duplicate messages. The default is 10 minutes.
Note:
The Inbound Document History (minutes) field can be set only if the Integration Server connects to a Broker version 6.0.1.
Inbound Client Side Queuing
Specify whether client-side queuing is enabled or disabled.
Select On if you want the server to place inbound guaranteed documents in document stores saved on disk. If the server shuts down unexpectedly before processing the document, the server recovers the document from the document store upon restart.
Select Off if you want the server to place guaranteed documents in document stores saved in memory. If the server shuts down unexpectedly before processing the document, the server recovers the document from the Broker.
Note:
Disable client-side queuing if the server is part of a cluster.
Note:
Client-side queuing is only available if the Integration Server connects to a 6.0.1 version of the Broker.
User
Specify the user account whose credentials the server uses to execute a service specified in a trigger condition for a webMethods messaging trigger that receive documents from the Broker. You can specify a predefined user account or a user account created by you or another server administrator. You can also select a user group/role from the central directory by clicking on the magnifying glass icon next to the user name. The default user is Administrator.
Administrator. Used to access the Integration Server Administrator to configure and manage the server.
Central. Used to monitor different servers from one Integration Server.
Default. Used when the client does not supply a user name and password.
Developer. Used to connect to the server from Software AG Designer to create, modify, and delete services that reside on the server.
Replicator. Used during package replication.
Outbound Document Store
Maximum Documents to Send per Transaction
Specify the number of documents that Integration Server Administrator should send from the outbound document store to the Broker in each transaction.
Note:
If there is no configured Broker, the Integration Server Administrator displays "Broker Not Configured" next to the field name.
Save Changes
Saves the specified settings. The settings with an asterisk (*) next to them will take effect when you restart Integration Server.
