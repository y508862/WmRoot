Cluster view
Use the Cluster View page to view trigger synchronization status across all the servers in the cluster. The Cluster view page indicates whether the other servers in the cluster (and its triggers) are in sync with current server.
Note:
The Cluster view page is only visible if you configured cluster synchronization. For more information about configuring cluster synchronization for triggers, see webMethods Integration Server Administrator’s Guide.
Click Return to Trigger Management to return to the display of trigger settings.
Cluster Server List
Use this section to view the trigger synchronization status across all the servers in the cluster. For each server listed in the watt.server.cluster.aliastList property, the cluster view indicates whether the other servers in the cluster (and its triggers) are in sync with current server.
If a trigger is not synchronized, the cluster view displays an error message that indicates how the trigger is out of sync with the trigger on the current server. For example, if document processing for a trigger is suspended locally, but active on another server in the cluster, the error message next to trigger name states "Processing State mismatch [local=suspended; remote=active]".
The Integration Server considers a trigger on a remote server to be out of sync with the local trigger of the same name if either of the following is true:
The triggers have different values for trigger queue capacity, refill level, or maximum execution threads.
The triggers have different document retrieval or document processing states.
Remote Server Alias
Specifies the remote server aliases for the servers in the cluster. Each server alias displayed in this column is listed in the watt.server.cluster.aliastList property.
To log on to a remote server in the cluster, click the server alias. When connecting, the remote server prompts you for user and password information. If you are connecting to the remote server via https and the https port requires certificates, you need to import a trusted certificate into the browser so that it can be presented at connection time. If the trusted certificates are not imported into the browser, when you try to connect to the remote server, you will receive a message informing you that the page is not available. For more information about client authentication and certificates, see webMethods Integration Server Administrator’s Guide.
Remote Server In Sync?
Indicates whether the triggers on the remote server are synchronized with the triggers on the current Integration Server. Yes indicates that all the triggers on the remote server use the same queue capacity, refill level, maximum execution threads, document retrieval state, and document processing state as corresponding triggers on the current Integration Server. No indicates that one or more of the triggers on the remote server are not synchronized with the corresponding triggers on the current Integration Server.
Trigger Name
Specifies the fully qualified name for the webMethods messaging trigger.
Trigger In Sync?
Indicates whether a trigger is in sync with the trigger of the same name on the current Integration Server. Yes indicates that the trigger uses the same values for queue capacity, refill level, maximum execution threads, document retrieval state, and document processing state. No indicates that the trigger on the remote server is not synchronized with the trigger of the same name on the current Integration Server.
Error Message
Displays an error message that identifies the mismatch between the trigger on the remote server and the trigger of the same name on the current Integration Server.
