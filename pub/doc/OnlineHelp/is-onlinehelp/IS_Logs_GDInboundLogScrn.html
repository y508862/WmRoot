Guaranteed delivery - Inbound
Use the Guaranteed delivery Inbound page to view the inbound and outbound transaction logs. The inbound transaction log (WMTXIN_yyyymmdd_hhmmss.log) records information about guaranteed-delivery requests that the server processes. The outbound transaction log (WMTXOUT_yyyymmdd_hhmmss.log) records information about guaranteed-delivery requests that the server makes to other webMethods Integration Servers. The logs contain a record of all critical operations that the server performs for each guaranteed delivery transaction.
You can use these logs with the appropriate job store to identify specific information about a transaction. For example, if a transaction fails, you can use the log to determine how far the transaction got before failing so you can take appropriate measures to recover it.
Guaranteed delivery - Inbound log columns
Note:
The Guaranteed delivery Inbound page might not display all of the columns listed below. Click > Show columns to hide or display columns for the inbound guaranteed delivery log.
Date/Time
Date and time the audit record was written to the database or file system.
Audit time stamp
Date and time the entry was written to the log.
Status
Status of service execution. The status can be either STARTED, ENDED or FAILED.
Message
The name of the process that is running.
Error Message
Optional error message, describing the type of error.
Server ID
IP address or host name of the server on which the Integration Server that generated the audit event is running. This is necessary information when Integration Servers are clustered and writing to a shared RDBMS. The ID can be DNSname:port or IPaddress:port.
Note:
The port is always the Integration Server’s primary port, even if the event occurred on a different (non‑primary) Integration Server port.
Root context
Context information that provides a consistent, interpretable audit trail of activity that the webMethods Monitor can use to relate entries within the log.
Parent context
Context information that provides a consistent, interpretable audit trail of activity that the webMethods Monitor can use to relate entries within the log
Current context
Context information that provides a consistent, interpretable audit trail of activity that the webMethods Monitor can use to relate entries within the log
