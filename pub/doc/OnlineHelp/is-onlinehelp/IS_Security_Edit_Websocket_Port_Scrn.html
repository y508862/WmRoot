Edit WebSocket port configuration
Use the Edit WebSocket port configuration page to view or specify the settings for a WebSocket port.
Click Return to Ports to return to the list of ports that are currently defined on the server.
Note:
You cannot edit port settings for the primary port.
WebSocket Listener Configuration
Use this section to specify general configuration information about this port.
Enable
Controls whether or not the port is enabled. When the port is enabled, it can receive requests.
Note:
A disabled port is not re-enabled by restarting the server. A disabled port does not accept requests until you explicitly enable it.
Port
Specifies the number of the port to which you want to assign a WebSocket listener. Type a port number that is not used by another port on your server.
Alias
An alias for the port. An alias must include one or more of the following: ASCII characters, numbers, underscore (_), period (.), and hyphen (-).
Description (optional)
A description of the port.
Package Name
Specifies the name of the package that you want to associate with this port. When you assign a package to a port, the server enables and disables the port in conjunction with the package. If you disable the port, the server disables the port automatically. A package that you assign to a port also retains its port assignment when it is published to other servers. If you do not have a package to assign to the port, leave the Package Name setting at WmRoot (the server's core package). This makes the port available whenever the server is running.
Bind Address (optional)
Optional. IP address to which to bind this port. Specify a bind address if your machine has multiple IP addresses and you want the port to use this specific address. If you do not specify a bind address, the server picks one for you.
Backlog
The number of requests that can remain in the queue for an enabled port before Integration Server begins rejecting requests. The default is 6000. The maximum value is 65535.
Note:
This parameter does not apply to disabled ports. Integration Server refuses requests sent to disabled ports.
Threadpool Configuration
Use this section to specify the minimum and maximum number of threads that are reserved for requests.
Threadpool Min
The minimum number of threads Integration Server maintains in this thread pool. The default is 10.
Threadpool Max
The maximum number of threads Integration Server maintains in this thread pool. The default is 200.
Thread Priority
Priority with which the Java treats threads from this thread pool. The larger the number, the higher the priority. The default is 5. Refers to the Java thread priority. The default is 5.
WebSocket Policy
Use this section to specify policy configuration for the WebSocket connections through WebSocket port.
Idle Timeout(ms)
Length of time, in milliseconds, the pool can keep an unused connection open. After the specified period of time elapses, the pool closes unused connections that are not needed to satisfy the Minimum Connections value.
Security Configuration
Client Authentication
Specifies the way in which you want the Integration Server to authenticate clients that submit requests through this port. Select one of the following:
Option
Description
Username/Password
The Integration Server prompts the client for a user ID and password.
Save Changes
Updates the Integration Server with the new or changed port settings. Your changes take effect immediately. When you view the port’s details, the server reports the total number of private threadpool threads currently in use for the port.
