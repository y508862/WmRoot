Use the Proxy servers page to view the current proxy server and proxy bypass settings. These settings specify the domains through which your server routes outbound requests. Also use this screen to edit, enable, disable, or delete proxy server aliases as required.
Click Create Proxy Server Alias to create new proxy server aliases and define their protocol and default properties.
Click Edit Proxy Bypass to add or modify the list of domains to which the Integration Server should issue an outbound request directly, bypassing the proxy.
Proxy Servers List
Displays information about the proxy servers to which Integration Server sends outbound HTTP, HTTPS, FTP, or SOCKS requests.
Default
Displays whether the displayed proxy server is the default proxy server for a particular protocol. Although you can have multiple proxy servers for each type of protocol - HTTP, HTTPS, FTP, or SOCKS- only one default can be selected for each protocol.
Alias
Displays the alias name of the proxy server. Alias names must be unique across protocols. In other words, the same alias cannot be repeated for a different protocol.
Protocol
Displays the type of protocol to be used for the proxy server. The protocol can be HTTP, HTTPS, FTP, or SOCKS.
Host
Displays the host name or IP address of the proxy server. You can either specify the proxy server's domain name or its numeric IP address.
Port
Displays the port on which the proxy server listens for HTTP, HTTPS, FTP and/or SOCKS requests
User
The user name Integration Server must use when accessing this proxy server. This is an optional feature.
Enabled
Displays if the proxy server is enabled or disabled.
Delete
Displays a red cross mark which enables you to delete the proxy server.
Proxy Bypass
Displays the names of servers to which the webMethods Integration Server issues outbound requests directly, bypassing the proxy server.
