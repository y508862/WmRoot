Broker connection alias details
Click Return to webMethods sessaging settings to return to the previous page.
Click Edit Broker connection alias to open the Edit Broker connection alias and edit the Broker connection alias details.
General Settings
Connection Alias Name
Name of the messaging connection alias.
Description
Description of the messaging connection alias.
Client Prefix
A string that identifies the Integration Server to Broker.The Broker Manager displays this prefix for each client it creates for the server. (The Broker creates multiple clients for each server that connects to it.)
Client Prefix Is Shared (prevents removal of shared messaging provider objects)
When selected, indicates that Integration Server shares the client prefix with multiple Integration Servers and you want to prevent automatic updates, including deletions, to shared objects associated with this alias on the Broker.
Connection Settings
Broker Host
Name (DNSname: port or ipaddress: port) of the machine on which the Broker Server resides.
Broker Name
Name of the Broker as defined on the Broker Server. The default name is Broker #1.
Client Group
Broker client group to which this Integration Server belongs. A client group defines a single set of properties and access permissions assigned to one or more clients (here, Integration Servers) on a single Broker.
Client Authentication
The type of authentication an Integration Server client will use to connect to the Broker.
Option
Description
None
Indicates that the Broker is configured to accept anonymous connections.
Username/Password
Indicates that the Broker uses basic client authentication. If you select this option, specify the user name and password the client will use in the Username and Password fields.
SSL
Indicates that Integration Server connects to the Broker using the SSL port. If you select this option, configure the SSL parameters by providing the following information.
Keystore
The full path to the keystore file for this Integration Server. A keystore file contains the credentials (private key/signed certificate) that an entity needs for SSL authentication. If the Broker requires an SSL connection, then the information in this file is used to authenticate the Integration Server client to that Broker Server.The keystore file for an Integration Server is stored on the machine on which the Integration Server resides. The Keystore parameter is displayed when Client Authentication is set to SSL.
Keystore Type
The file type of the keystore file for Integration Server. The file type can be either PKCS12 or JKS. The Keystore Type parameter is displayed when Client Authentication is set to SSL.
Keystore Password
Password required to access the SSL certificate in the keystore file for Integration Server. The Keystore Password parameter is displayed when Client Authentication is set to SSL.
Encryption Settings
Encryption
Specifies whether or not the connection between Integration Server and Broker is encrypted.
Truststore
The full path to the truststore file for the Integration Server client. A truststore file contains trusted root certificates for the authorities responsible for signing SSL certificates. For an SSL connection to be made, a valid trusted root for the SSL certificate stored in the keystore must be accessible in a truststore file. The truststore file for Integration Server is stored on the machine on which the Integration Server resides. The Truststore parameter appears only when for the Encryption parameter is set to Yes.
Truststore Type
The file type of the truststore file for Integration Server. The file type is JKS. The Truststore parameter appears only when for the Encryption parameter is set to Yes.
Note:webMethods Broker is deprecated.
