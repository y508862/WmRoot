Edit WS endpoint trigger
Use the Edit WS (web service) endpoint trigger details page to edit message processing properties for a WS endpoint trigger.
Click Return to JMS Trigger Management to return to the page for managing all JMS triggers.
Click Return to Trigger Detail to return to the previous page.
Properties
Use this section to change the message processing properties for the WS endpoint trigger.
JMS Connection Alias
Select the JMS connection alias that you want the WS endpoint alias trigger to use to retrieve messages from the JMS provider.
Destination Name
Destination from which you want the WS endpoint trigger to receive messages.
Processing Mode
Specifies the processing mode for the WS endpoint trigger.
Select...
To...
Serial
Specify that Integration Server process messages received one after the other in the order received.
Concurrent
Specify that Integration Server process multiple messages at one time.
Max Execution Threads
Specify the maximum number of messages that can be processed by Integration Server for this trigger at one time. For serial triggers, Max Execution Threads must be equal to 1.
Connection Count
Specify the maximum number of connections to the JMS provider that Integration Server will make for this trigger. This property applies to concurrent JMS triggers only.
Save Changes
Saves the changes to the WS endpoint trigger.
