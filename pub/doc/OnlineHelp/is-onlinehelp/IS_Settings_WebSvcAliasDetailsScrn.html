Web service alias details
Use the Web Service Alias Details page to view the transport, security, and message addressing properties of a web service alias on the Integration Server.
Click Return to Web Service Endpoints to return to the previous screen.
Click Edit Web Service Endpoint Alias to change the properties of a web service endpoint alias. For detailed information about changing alias properties, see Edit alias .
Web Service Endpoint Alias Properties
Value
Description
Alias
The name of the alias.
Description
Description of the alias.
Type
Whether this endpoint alias is for use with a provider web service descriptor or a consumer web service descriptor.
Transport Type
The transport protocol used to access the web service.
Execute ACL
The ACL that governs which user groups on your server can use this web service endpoint alias. By default, only members of groups governed by the Internal ACL can use this alias.
This field is available only for consumer endpoint aliases.
Note:
Integration Server performs the ACL check only if the specific endpoint alias is used as the value of the endpointAlias input parameter in a web service connector or the pub.client:soapClient service. Integration Server does not perform the ACL check when the consumer web service endpoint alias is assigned to a binder used by the web service connector.
HTTP Transport Properties
Transport properties are used to authenticate the credentials or certificate in the HTTP header of requests, for web services located at this endpoint. Different properties will be displayed, depending on the combination of Type and Transport Type.
Value
Description
Host Name or IP Address
The host name or IP address of the server on which the provider web service exists.
Port Number
The port number associated with the alias. This is the port number to which requests for the provider web service should be sent.
Authentication Type
The type of authentication to use to authenticate the credentials. This field is available for consumer and message addressing endpoint aliases.
Value
Description
Basic
Use basic authentication (user name and password) to authenticate the credentials.
Digest
Use password digest to authenticate the credentials.
NTLM
Use NTLM authentication so that clients that are already logged into a domain can be authenticated using their existing credentials.
Kerberos
Use Kerberos authentication to authenticate the credentials.
User Name (optional)
The user name that must be included in the HTTP header of the web service request, if the web service provider requires basic authentication in the HTTP header.
Proxy Alias
The proxy alias for the proxy server through which web service requests must be sent. You need to specify a proxy alias only if the Integration Server on which the web service connectors resides is behind a firewall. You can only specify a proxy alias for a consumer web service descriptor.
Keystore Alias
The text identifier for the keystore, which contains private keys and certificates for an Integration Server, partner application, or Integration Server component.
Key Alias
The text identifier for the private key associated with the keystore alias.
Kerberos Transport Properties (Optional)
Kerberos properties are used to enable Kerberos authentication at transport-level. You can provide Kerberos-related details that will be used for all web service requests or providers that use this endpoint alias.
These fields are available for consumer endpoint aliases using HTTP and HTTPS transport type.
Field
Description
JAAS Context
Specify the custom JAAS context used for Kerberos authentication.
In the following example, JAAS Context is KerberosClient:
KerberosClient {
com.sun.security.auth.module.Krb5LoginModule required useKeyTab=true keyTab=alice.keytab;
};
Principal
Specify the name of the principal to use for Kerberos authentication.
Principal Password
Specify the password for the principal that is used to authenticate the principal to the KDC. Specify the principal password if you do not want to use the keytab file that contains the principals and their passwords for authorization. The passwords may be encrypted using different encryption algorithms.
If the JAAS login context contains useKeyTab=false, you must specify the principal password.
Retype Principal Password
Re-enter the above principal password.

WS Security Properties (Optional)
WS-Security properties are used by the SOAP processor to provide security information in the WS-Security header of the SOAP message. Different properties will be displayed, depending on the Type in the Web Service Endpoint Alias Properties section.
Value
Description
User Name
Name to include with the Username Token, if the web service's security policy requires one.
Partner's Certificate
Path and file name of the file that contains the partner's certificate, for example d:/certs/partnerkey.der. or config/certs/ partnerkey.der.
Keystore Alias
The text identifier for the keystore, which contains private keys and certificates for an Integration Server, partner application, or Integration Server component.
Key Alias
The text identifier for the private key associated with the keystore alias.
Truststore Alias
The text identifier for the truststore, which contains the trusted root of a certificate or signing authority (CA).
Timestamp Precision
Whether the timestamp placed in the Timestamp element of the security header of an outbound message is precise to seconds or milliseconds. If the precision is set to milliseconds, Integration Server uses the timestamp format yyyy-MM-dd'T'HH:mm:ss:SSS'Z'. If the precision is set to seconds, Integration Server uses the timestamp format yyyy-MM-dd'T'HH:mm:ss'Z'.
Integration Server uses the timestamp precision value only when WS-Security is implemented via a WS-Policy.
Timestamp Time to Live (seconds)
The time-to-live value for an outbound message in seconds. Integration Server uses this value to set the expiry time in the Timestamp element of outbound messages.
Integration Server uses the timestamp precision value only when WS-Security is implemented via a WS-Policy.
Timestamp Maximum Skew (seconds)
The maximum number of seconds that the web services client and host clocks can differ so that the timestamp expiry validation does not fail.
Integration Server uses the timestamp precision value only when WS-Security is implemented via a WS-Policy. Integration Server validates the inbound SOAP message only if the creation timestamp of the message is less than the sum of the timestamp maximum skew value and the current system clock time.

Message Addressing Properties of a Provider Web Service Endpoint Alias (Optional)
Message addressing properties contain the WS-Addressing information that Integration Server uses to generate the WS-Addressing headers of the SOAP requests and responses.
Value
Description
To
URI of the destination of the SOAP message.
The Reference Parameters field displays the additional parameters, if any, that correspond to <wsa:ReferenceParameters> properties of the endpoint reference to which the message is addressed. The Metadata Elements field displays optional metadata (such as WSDL or WS-Policy) about the service. The Extensible Elements field displays elements other than those specified as part of the Metadata and Reference Parameters.
Response Map
Address to which the provider sends the reply or fault message and the corresponding message addressing alias. Integration Server retrieves the authentication details needed to send the response from the message addressing alias mapped to the address.
Message Addressing Properties of a Consumer or a Message Addressing Endpoint Alias (Optional)
Message addressing properties contain the WS-Addressing information that Integration Server uses to generate the WS-Addressing headers of the SOAP requests and responses.
Value
Description
Must Understand
Whether the recipients (the actor or role to which the header is targeted) are required to process the WS-Addressing headers. Recipients that cannot process a mandatory WS-Addressing header reject the message and return a SOAP fault.
Must Understand determines the mustUnderstand attribute of the WS-Addressing headers.
Value
Description
True
Indicates that processing the WS-Addressing headers is required by the recipients.
If you select True for Must Understand and the SOAP node receives a header that it does not understand or cannot process, it returns a fault.
False
Indicates that processing the WS-Addressing headers is optional. This is the default.
Note:
In SOAP 1.1, the values of the mustUnderstand attribute were 0 and 1 instead of True and False; however, Integration Server processes both sets of values the same way and performs any necessary conversions. For more information about the mustUnderstand and actor attributes in SOAP 1.1, see the Simple Object Access Protocol (SOAP) 1.1 - W3C Note 08 May 2000 at http://www.w3.org/TR/SOAP/.For more information about the mustUnderstand and role attributes in SOAP 1.2, see the Simple Object Access Protocol (SOAP) 1.2 specification at http://www.w3.org/TR/SOAP12/.
Role
Target of the WS-Addressing headers in the SOAP message. Role determines the value of the role attribute for the WS-Addressing headers. The actor or role attribute specifies a URI for the recipient of WS-Addressing header entries.
Note:
In SOAP 1.1, the role attribute is named actor; however, Integration Server processes both names the same way and performs any necessary conversions.
Value
Description
Ultimate Receiver
Indicates that the recipient is the ultimate destination of the SOAP message. This is the default.
Next
Indicates the following URI for the role attribute:
For SOAP 1.2: "http://www.w3.org/2003/05/soap-envelope/role/next"
For SOAP 1.1: "http://schemas.xmlsoap.org/soap/actor/next"
None
Indicates the following URI for the role attribute:
For SOAP 1.2: "http://www.w3.org/2003/05/soap-envelope/role/none"
For SOAP 1.1: "http://www.w3.org/2003/05/soap-envelope/role/none"
Other
Specifies the target of the header. Typically, this will be a URI.

Reliable Messaging Properties of a Provider or Consumer Endpoint Alias (Optional)
Reliable messaging properties provide information specific to the web service endpoint. By default, web service endpoints use the reliable messaging properties defined for Integration Server on the Reliable Messaging Properties (Settings > Web Services > Reliable Messaging > Edit Configuration) page. You can override the server-level reliable messaging configuration with reliable messaging properties specific to the web service endpoint alias.
Under Reliable Messaging Properties, if Enable is true, it specifies that Integration Server uses the following reliable messaging properties specific to the web service endpoint alias.
Field
Description
Retransmission Interval
The time interval (in milliseconds) for which a reliable messaging source waits for an acknowledgement from the reliable messaging destination before retransmitting the SOAP message. The default is 6000 milliseconds.
Acknowledgement Interval
The time interval (in milliseconds) for which the reliable messaging destination waits before sending an acknowledgement for a message sequence. Messages of the same sequence received within the specified acknowledgement interval are acknowledged in one batch. If there are no other messages to be sent to the acknowledgement endpoint within the specified acknowledgement interval, the acknowledgement is sent as a stand-alone message.
The default is 3000 milliseconds.
Exponential Backoff
Whether to use the exponential backoff algorithm to adjust the retransmission interval of unacknowledged messages. Adjusting the time interval between retransmission attempts ensures that a reliable messaging destination does not get flooded with a large number of retransmitted messages.
Value
Description
true
Increases the successive retransmission intervals exponentially, based on the specified retransmission interval. For example, if the specified retransmission interval is 2 seconds, and the exponential backoff value is set to true, successive retransmission intervals will be 2, 4, 8, 16, 32, and so on if messages continue to be unacknowledged. This is the default.
false
Uses the same time interval specified in the Retransmission Interval field for all retransmissions.
Maximum Retransmission Count
The number of times the reliable messaging source must retransmit a message if an acknowledgement is not received from the reliable messaging destination. To specify that there is no limit to the number of retransmission attempts, set the value of Maximum Retransmission Count to -1. The default is 10.
