Password Expiration Settings
Use the Password Expiration Settings page to view the criteria for password expiry.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Click Edit Password Expiration Settings to change the password expiry settings.
Password Expiration Settings
Use this section to specify the criteria for password expiration. You also use this page to specify whether to enable the expiry settings.
Enabled
Indicates whether or not password expiration settings are in use.
Expiration Interval
Specifies the number of days after which a password will expire, if the password is not changed. The value should be a non-zero integer. Default is 90 days.
Expiration Notice Email Addresses
Specifies the list of email addresses to which Integration Server sends an email notification informing that the user password is about to expire or has already expired. If the field is blank, then it indicates that Integration Server uses the email address specified in the Internal Email field of Email Notification section on Resources page ( Settings > Resources).
Note:
Integration Server uses the SMTP server and port details specified in the Email Notification section on Resource Settings page (Settings > Resources).
Expiration Email Reminders
Specifies the number of days prior to password expiry that Integration Server starts sending the reminder emails for password reset. Integration Server sends emails until the password is either updated or the expiration interval is changed. Set the value to 0 to prevent Integration Server from sending the reminder emails for expired or soon to be expired passwords. Default is 3.
Applies to Users
Specifies the list of users to whom the expiration interval setting is applicable.
