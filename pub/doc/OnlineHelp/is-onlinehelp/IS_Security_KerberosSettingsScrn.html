Kerberos
 
Edit Global Claim Settings
The Kerberos Settings page displays the settings that Integration Server requires to enable Kerberos authentication for web service requests.
Click Edit Kerberos Settings to specify or edit the Kerberos settings.
Realm
Specifies the domain name of the Kerberos server, in all uppercase letters. All the computers managed by the KDC and secondary KDCs, if any, constitute the realm. That is, the realm includes all the nodes that share the same Kerberos database.
Example: KERBEROS.RNDLAB.LOC
Key Distribution Center Host
Specifies the host name of the machine on which the KDC resides.
Example: lab.kerberos.rndlab.loc
Kerberos Configuration File
Specifies the location of the Kerberos configuration file that contains the Kerberos configuration information, including the locations of KDCs, defaults for the realm and for Kerberos applications, and the hostnames and Kerberos realms mappings.
Important:
Values specified for Realm and Key Distribution Center Host overwrite the default KDC and realm set in the KDC configuration file specified in Key Distribution Center Host. Do not specify values for Realm and Key Distribution Center Host if you do not want to overwrite the default KDC and realm in the KDC configuration file.
Use Subject Credentials Only
Specifies whether Integration Server requires a Kerberos V5 Generic Security Services (GSS) mechanism to obtain the necessary credentials from an existing subject set up by the JAAS authentication module. Here, “subject” represents the user or service being authenticated in the JAAS login context.
When the Use Subject Credentials Only check box is selected, Integration Server requires a GSS mechanism to obtain the credentials from an existing Subject. Integration Server uses the Ticket Granting Ticket (TGT) stored in the subject to establish a GSS security context. The service ticket is also stored in the subject. When the Use Subject Credentials Onlycheck box is selected, the JVM in which Integration Server runs can use only the credentials found in the Subject in the JAAS authentication module. The JVM cannot use another underlying mechanism to obtain the credentials.
When the Use Subject Credentials Only check box is cleared, Integration Server does not require a GSS mechanism to obtain credentials from an existing Subject. Instead, the JVM in which Integration Server runs can use another underlying mechanism, such as a reading from a protected file on disk, to obtain credentials for the Subject.
The Use Subject Credentials Only check box must be selected if you want to use Kerberos authentication for web service requests.
