Use the Ports page to view the current port assignments for the server.
Click Add Port to configure a new port assignment.
Click Change Primary Port to select a new primary port.
Click Change Global IP Access Restrictions to specify how you want your server to filter inbound requests on ports whose access restrictions are not explicitly defined.
Port List
This section displays the list of ports that are configured on your server.
Port
Displays numbers of the ports that are configured on your server. To view detailed information about a particular port, click its port number.
Alias
An alias for the port. An alias must include one or more of the following: ASCII characters, numbers, underscore (_), period (.), and hyphen (-).
Protocol
Displays the protocol that this port uses to interact with clients. Will display HTTP, HTTPS, FTP, FTPS, File Polling, and Email protocols. In addition, the Integration Server also provides a diagnostic port and ports that webMethods Enterprise Gateway uses.
Type
Displays the purpose of the port. Will display one of the following:
Type
Description
Diagnostic
A special port that uses threads from a dedicated thread pool to accept requests for services via HTTP.
Enterprise Gateway External
A port used in a webMethods Enterprise Gateway configuration. This port exists on the Enterprise Gateway Server and communicates with external clients.
Enterprise Gateway Registration
A port used in a webMethods Enterprise Gateway configuration. This port exists on the Enterprise Gateway Server and communicates with the Internal Server.
Primary
The port that currently functions as the server’s primary port (designated by a green check mark). The primary port is an HTTP or HTTPS port that you designate as the server’s main listening port. The server uses this port in the same way as other HTTP or HTTPS ports, except that it doesn’t allow a primary port to be deleted. The primary port number is also the number that is reflected in the server’s watt.server.port setting. This is the number that clients will receive when they query your server’s watt.server.port property. To switch the primary port, click the Change Primary Port link.
Registration Internal
A port used in a webMethods Enterprise Gateway configuration. This is the port on the Enterprise Gateway to which the Internal Server connects, as viewed from the Internal Server.
Regular
A port that uses the HTTP, HTTPS, FTP, FTPS, Email, or File Polling protocol to accept requests from webMethods and other clients.
Package
Displays the name of the package associated with this port. When you enable the package, the server enables the port. When you disable the package, the server disables the port. When you associate a package with a port, the package will retain its port assignment when it is published to other servers (that is, when you replicate this package to another server, a port with this number and the same settings is created automatically on the target server).
Enabled
Displays the status of the port. When the port is enabled, the server can accept requests on that port. You may block incoming requests on a port by disabling it. To disable a port, click the Yes link in this column. To re-enable a port, click the No link in this column.
Note:
A disabled port is not re-enabled by restarting the server. A disabled port does not accept requests until you explicitly enable it.
Access Mode
Displays the type of access clients have to services and web service connectors through this port.
Status
Description
Allow
Clients are allowed to access all services and web service descriptors.
Allow +
Clients are allowed to access all services and web service descriptors except those specified in the Deny List.
Deny
Clients are not allowed to access any services or web service descriptors.
Deny +
Only IP addresses specified in the Allow List are allowed to connect to this port.
Click the Allow or Deny link in this column to view and/or edit the list of services that a user can request through the port. If a client requests a service that the port does not allow, the request is denied.
IP Access
Displays information about which IP addresses are allowed to connect to this port.
Status
Description
Allow
All IP addresses are allowed to connect to this port.
Allow +
All IP addresses, except those specified in the Deny List, are allowed to connect to this port.
Deny
No IP addresses are allowed to connect to this port.
Deny +
Only IP addresses specified in the Allow List are allowed to connect to this port.
Click the Allow or Deny link in this column to view and/or edit the list of host servers that are permitted to make requests through the port. If the port receives a request from a host that is not unauthorized to use it, the request is denied.
Advanced
Click the Edit link in this column to view and/or edit the rate at which the listener accepts connections and controls on this port, over the size of the threadpool (if it was enabled). With this Advanced Controls feature, you can define a delay value or specify thread pool controls to manage a large volume of simultaneous requests on the server.
Delete
Deletes the port specified in Port. To delete a port, click the icon in the port's Delete column.
Description
A description of the port.
