Reset All Outbound Passwords
If you lose or forget the master password or it becomes corrupted, use the Reset All Outbound Passwords page to reset all the stored outbound passwords and the master password. This procedure clears (blanks out) the stored outbound passwords and resets the master password to "manage." In addition, you must manually re-enter the appropriate passwords for all application and subsystem passwords on their respective configuration pages in the Integration Server Administrator.
Important:
Reset the outbound passwords only if your Integration Server encrypts outbound passwords using PBE and you have lost your master password. Once you reset the passwords, the Integration Server will no longer be able to connect to the applications and subsystems for which the passwords are required, until you manually reset them.
Reset Passwords
Resets the master password to "manage" and deletes all outbound passwords. Once you have reset the passwords, you must do the following:
1. From the Update Master Password page change the master password something other than “manage.”
2. Go to the configuration page for each application or subsystem and re-enter the password required for the Integration Server to connect to that application or subsystem. Pages to check include those that define:
Remote Integration Servers
Clustered servers
JDBC connection pools
POP/IMAP servers
LDAP servers
Proxy servers
Internal Server Port
Email Port
Databases
Keystores
Web Services Endpoint Aliases
