Set ACL
Use the Set ACL page to change the Access Control List (ACL) assignments of a folder or service. (You can also change this setting with Software AG Designer.) To change ACL settings for other elements, such as Schemas and Triggers, you must use Software AG Designer.
Click Return to browsing folderName to return to the parent folder.
Target
Displays the name of the resource (folder or service) whose ACL setting you want to change.
List ACL
The List ACL controls who can list a resource on screens in Designer and Integration Server Administrator. List access also allows a user to view an element's metadata. Resources that do not have a specific List ACL assignment will inherit one from their parent folder. Angle brackets <> around an ACL name indicate that the ACL setting is inherited. To change the List ACL assigned to the resource specified in Target, select a new ACL from the list.
Read ACL
The Read ACL has no meaning for a folder itself, but resources contained in the folder will inherit this setting for Read ACL if they have not already been assigned one. For services, the Read ACL controls who can view the main source through Designer and Integration Server Administrator. Angle brackets <> around an ACL name indicate that the Read ACL setting is inherited. To change the Read ACL assigned to the resource specified in Target, select a new ACL from the list.
Write ACL
For folders, the Write ACL controls who can add, edit, and delete resources in the folder. The Write ACL also controls who can change ACL assignments for the folder. Resources contained in the folder will inherit this setting for Write ACL if they have not already been assigned one. For services, the Write ACL controls who can edit the service. Angle brackets <> around an ACL name indicate that the Write ACL setting is inherited. To change the Write ACL assigned to the resource specified in Target, select a new ACL from the list.
Execute ACL
For folders, the Execute ACL has no meaning for the folder itself, but resources contained in the folder will inherit this setting for Execute ACL if they have not already been assigned one. For services, the Execute ACL controls who can execute the service. This access also gives the user access to files the server serves, such as DSP and .htm files. Angle brackets <> around an ACL name indicate that the folder inherits its Execute ACL setting from the parent folder. To change the Execute ACL assigned to the resource specified in Target, select a new ACL from the list.
Enforce ACL on Internal Invokes
(Displayed for services only.) Specifies whether access to this service is controlled on internal invokes (when the service is invoked internally by another service) and external invokes (when the service is requested by an external client). When a service is externally invoked, the server always performs an ACL check to verify that the requester is authorized to invoke that service. However, it does not perform an ACL check on internally invoked services unless the Enforce ACL on Internal Invoke option is enabled on the service being invoked.
Save Changes
Applies the ACL settings to the resources specified in Target. The changes take effect immediately.
