Universal Messaging connection alias details
Click Return to webMethods Messaging Settings to return to the previous page.
Click Edit Universal Messaging Connection Alias to open the Edit Universal Messaging connection alias and edit the Universal Messaging connection alias details.
General Settings
Connection Alias Name
Name of the messaging connection alias.
Description
Description of the messaging connection alias.
Client Prefix
A string that identifies the Integration Server to Universal Messaging. If your Integration Server belongs to a cluster, make sure all servers in the cluster use the same client prefix.
Client Prefix Is Shared (prevents removal of shared messaging provider objects)
When selected, indicates that Integration Server shares the client prefix with multiple Integration Servers and you want to prevent automatic updates, including deletions, to shared objects associated with this alias on Universal Messagingserver.
Connection Settings
Realm URL
The URL for the Universal Messaging server in the format nsp:// UM_host: UM_port (for example, nsp://127.0.0.1:9000 or nsp://localhost:9000).
Maximum Reconnection Attempts
Specify the maximum number of reconnection attempts that Integration Server makes if the connection to Universal Messaging fails. If the connection cannot be re-established, Integration Server writes messages to the error log and the connection created by this messaging connection alias will be stopped. The default is 5 attempts.
Producer Settings
Enable CSQ
Whether a client side queue is used with this Universal Messaging connection alias.
Yes indicates a client side queue is used with the alias. When a client side queue is in use, if the Universal Messaging server is not available when Integration Server publishes documents using this Universal Messaging connection alias Integration Server writes the documents to the client side queue.
No indicates a client side queue is not used. When a client side queue is not in use, if the Universal Messaging server is not available when Integration Server publishes documents using this Universal Messaging connection alias, the publishing service ends with an ISRuntimeException
Maximum CSQ Size (messages)
The maximum number of documents (messages) that can exist in the client side queue for this Universal Messaging connection alias. A value of UNLIMITED indicates the client side queue can contain an unlimited number of messages.
Drain CSQ in Order
Whether Integration Server drains the client side queue for this alias by sending the messages to the Universal Messaging server in the same order in which Integration Server placed the messages in the client side queue.
Yes indicates the client side queue is drained in order. After the connection to the Universal Messaging server is re-established, Integration Server continues to write new messages to the client side queue until the client side queue is completely drained.
No indicates that the client side queue is not drained in order. After the connection to the Universal Messaging server re-established, Integration Server sends new messages directly to the Universal Messaging server while it drains the client side queue
Publish Wait Time while Reconnecting (milliseconds)
Number of milliseconds that a publishing service using this Universal Messaging connection alias will wait for a connection to the Universal Messaging server to be re-established after the connection fails. If Integration Server re-establishes the connection before the Publish Wait Time While Reconnecting elapses, the publishing service continues executing. If the specified time elapses before a connection is re-established, the publishing service ends with an ISRuntimeException. The default is 0 milliseconds, which means that publishing services will not wait for Integration Server to re-establish the connection.
Enable Follow the Master for Producers
Indicates whether Integration Server always connects to the master realm server in a Universal Messaging cluster when Integration Server uses this Universal Messaging connection alias to send messages.
Consumer Settings
Enable Request/Reply Channel and Listener
Indicates whether a request/reply channel is created for this Universal Messaging connection alias on Universal Messaging and whether a listener for replies is created on Integration Server.
Enable Follow the Master for Consumers
Indicates whether Integration Server always connects to the master realm server in a Universal Messaging cluster when Integration Server uses this Universal Messaging connection alias to retrieve messages.
Client Authentication Settings
Client Authentication
Type of authentication that an Integration Server client will use to connect to a Universal Messaging server. None indicates that no authentication will be performed. Username/Password indicates that basic client authentication will be performed using user name/password combinations. Certificate Based indicates that Universal Messaging requires certificates for authentication.
Truststore Alias
The alias for the truststore that contains the certificate authority (CA) certificates for the Universal Messaging server. You must select a truststore alias if the Universal Messaging port is configured for SSL, one-way authentication, or two-way authentication.
Keystore Alias
The alias for the keystore that contains the client certificates that you want Integration Server to use when connecting to the Universal Messaging port. You must select a keystore alias if the Universal Messaging port is configured for two-way authentication.
Key Alias
The alias to the key that contains the private key for connecting to the Universal Messaging port securely. The key alias must be in the keystore specified in Keystore Alias.
Enhanced Logging
Logging Type
Specifies where Integration Server writes log messages when enhanced logging is enabled for the message producers and/or consumers that use this messaging connection alias to send and/or receive documents.
Specify...
To...
SERVER LOG
Write enhanced logging messages to the server log. If you specify the server log as the destination, make sure to increase the logging level for the 0168 Messaging (Enhanced Logging) server log facility to at least Info.
MESSAGING AUDIT LOG
Write enhanced logging messages to the messaging audit log.
Enable Producer Message ID Tracking
When set to Yes, indicates that Integration Server writes additional log messages when a message producer uses this messaging connection alias to publish documents that are instances of the publishable document types listed in Producer Message ID Tracking: Included Channels.
Producer Message ID Tracking: Included Channels
The name of each channel that corresponds to a publishable document type for which Integration Server performs additional logging during publication. This field displays [ALL] if Integration Server performs enhanced logging for every channel to which this messaging connection alias sends messages.
Enable Consumer Message ID Tracking
When set to Yes, indicates that Integration Server writes additional log messages for messaging consumers (triggers) that use this messaging connection alias to receive messages. Integration Server writes additional log message for the triggers listed in Consumer Message ID Tracking: Included Triggers.
Consumer Message ID Tracking: Included Triggers
The fully qualified name of the triggers for which Integration Server performs additional logging during trigger processing. This field displays [ALL] if Integration Server performs enhanced logging for every trigger that uses this messaging connection alias to receive messages.
