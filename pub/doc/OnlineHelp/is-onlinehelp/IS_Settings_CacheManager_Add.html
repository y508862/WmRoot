Add cache manager
Use the Add cache manager page to create a cache manager using Integration Server Administrator.
Click Return to Caching to return to the Caching page.
Cache Manager Configuration
Name
Name of the cache manager. A valid cache manager name:
Must not be null.
Must not contain these characters: ? [ ] / \ = + < > : ; " , * | ^ @
Must not start with a period (.) or end with a space.
Must not begin with the characters SoftwareAG.
Must not be a device name in any operating system.
Note:
You cannot modify the name of a cache manager once the cache manager is created.
Terracotta Server Array URLs
Comma-separated list of host names and port numbers, one for each server in the Terracotta Server Array. You can add multiple URLs by using this format: host1:port,host2:port,...
Rejoin
Whether a node in a distributed cache must automatically reconnect to the Terracotta Server Array upon getting disconnected. If you enter host names and port numbers in the Terracotta Server Array URLs field, the Rejoin check box will be automatically selected.
After the node reconnects to the Terracotta Server Array, the disconnected node is discarded and a new node is created. The new node will initially be empty, but will connect to the Terracotta Server Array and pull the server cache entries on demand.
Save Changes
Saves the cache manager configuration.
Note:
Integration Server does not start the cache manager automatically after creating it. You must start the cache manager by clicking the Start link.
