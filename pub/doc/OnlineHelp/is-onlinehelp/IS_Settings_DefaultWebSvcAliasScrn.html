Set default provider endpoint aliases
Use the Set default provider endpoint aliases page to specify a default provider endpoint alias for the HTTP and/or HTTPS protocols. When a binder in a provider web service descriptor uses the default alias, Integration Server uses the information in the default alias when constructing the WSDL for the descriptor. Changing the default alias for a protocol changes the information used to generate the WSDL document for web service descriptors that rely on the default provider endpoint alias for that protocol.
Click Return to Web Services to return to the previous screen.
Select Default Provider Aliases
HTTP
Select the provider endpoint alias to use as the default provider endpoint alias for the HTTP protocol. If you do not want to set a default endpoint alias for HTTP, select the blank row.
HTTPS
Select the provider endpoint alias to use as the default provider endpoint alias for the HTTPS protocol. If you do not want to set a default endpoint alias for HTTPS, select the blank row.
Update
Updates the default aliases.
