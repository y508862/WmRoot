The Enterprise Gateway rules page displays a list of Enterprise Gateway rules. Use this page to create, view, edit, copy, delete, enable, or disable an Enterprise Gateway rule. You can also use this page to change a rule’s alert options or priority.
Important:
Enterprise Gateway runs on an Integration Server that has been configured as an Enterprise Gateway Server running in your DMZ. The Enterprise Gateway Server requires a webMethods Enterprise Gateway license.
Note:
This page is disabled if webMethods API Gateway is installed on Integration Server.
Click Default Alert Options to view and edit the default alert options for Enterprise Gateway rules.
Click Denial of Service Options to set request limits to prevent Denial of Service attacks.
Click Mobile Application Protection Options to specify mobile device types and mobile applications from which you want to restrict requests.
Click Create Rule to create new Enterprise Gateway rules.
Denial/Alert Rules
The Denial Rules section displays the rules that enforce the Deny Request and Alert action. The Alert Rules section displays the list of rules that enforce the Alert action.
Rule Name
Name of the rule. Click the underlined name of a rule to view or change details about a rule.
Filters
One or more filters that the rule applies to requests. Depending on the type of rule, if a request meets the conditions of the filter, Enterprise Gateway Server either denies the request and sends an alert about the denial or simply sends an alert.
Enabled
Whether the rule is currently enabled or disabled.
Alert Options
Whether the default alert option is applied to the rule. If the rule uses the default alert option, Enterprise Gateway Server displays Default in this column. If the alert option is specific for the rule, the server displays Custom in this column. Click the Default or Custom link to view or edit the alert options.
Copy
Creates a copy of the rule. You can create a new Enterprise Gateway rule by copying an existing rule. When you copy a rule, the filter settings, alert options, and other properties, except for rule name, are copied.
Delete
Deletes the rule.
Priority
Enables you to change the order in which Enterprise Gateway Server enforces the rules on requests.
