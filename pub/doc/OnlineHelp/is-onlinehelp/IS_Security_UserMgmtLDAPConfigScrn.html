LDAP Configuration
Use the LDAP Configuration page to configure the server to access user account and group information from an external directory. You can configure the server to use Lightweight Directory Access Protocol (LDAP).
Click Return to User management to return to the list of groups and users that are currently defined on your server.
Click Edit LDAP Configuration to configure an external directory server or change the current settings. The current configuration is displayed on the LDAP Configuration section.
Click Add LDAP Directory to add multiple LDAP servers. The current list of LDAP directories are displayed with the option to delete a URL if it needs to be removed. This option is only displayed when the Provider field is set to LDAP.
LDAP Configuration
To open the LDAP Configuration page, click Edit LDAP Configuration. Choose a Provider to specify the external directory that you want to use.
Provider
Specifies the type of external directory server, if any, that this server uses for user account and group information. Choose Local or LDAP. Only one feature can be enabled at a time. The server issues a prompt to verify that you want to change the setting. Click OK. Note that LDAP and CDS parameters are exclusive of each other - when CDS is enabled (Local), LDAP is disabled and vice versa. The remaining parameters on this page will vary depending on which provider you use. See the sections below on LDAP Settings for specific descriptions.
Cache Size (Number of Users)
Specifies the maximum number of LDAP users the Integration Server can keep in memory in the user cache. The default is 10.
Once the limit is reached, the Integration Server selects users for removal from the cache based on how long they have been idle. As a result, activity can extend the time a user remains in the cache.
As a general rule, specify 5-10% of the number of users in your LDAP system. However, if only a few sessions are ever logged on simultaneously, set the cache size to be the same as the number of simultaneous sessions.
Credential Time-to-Live (Minutes)
Shows the number of minutes an LDAP user's credentials (user ID and password) can remain in the credential cache before being purged. The default is 60 minutes.
When a user first attempts to log in, the Integration Server creates a user object and checks the user's credentials against the LDAP directory. The Integration Server stores the credentials so that subsequent requests to authenticate will be made against the cached credentials, not the LDAP directory. For security reasons, you can control the length of time these cached credentials are valid.
Save Configuration
Save the changes that you specified. Your changes will take effect when you restart the server.
