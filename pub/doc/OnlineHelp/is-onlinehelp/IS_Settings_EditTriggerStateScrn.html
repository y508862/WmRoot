Edit all triggers
Click Return to Trigger Management to return to the display of trigger settings.
State
Retrieval State
Specifies the document retrieval state for all the webMethods messaging triggers on Integration Server. Select Active to make document retrieval active for all triggers. Select Suspended to suspend document retrieval for all the triggers.
Processing State
Specifies the document processing state for all the webMethods messaging trigger on the Integration Server. Select Active to make document processing active for all triggers. Select Suspended to suspend document processing for all the triggers on the Integration Server.
Change Mode
Apply Change Permanently
When selected, indicates that thee state change is permanent and will be maintained after Integration Server restarts, the trigger is enabled or disabled, or the package containing the trigger reloads.
Apply Change Across Cluster
Select this check box if you want to apply the document retrieval and processing state changes to all the Integration Servers in a cluster.
Note:
The Apply Change Across Cluster check box appears only if the current Integration Server belongs to a properly configured cluster and is configured to synchronize trigger changes across the cluster.
Save Changes
Saves the specified settings. Server restart is not required for changes to take effect.
