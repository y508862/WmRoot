Default Alert Options
Use the Default Alert Options page to view the settings for the default alert options for Enterprise Gateway rules. Enterprise Gateway Server uses the default alert options when a rule does not specify alert options of its own.
Important:
Enterprise Gateway runs on an Integration Server that has been configured as an Enterprise Gateway Server running in your DMZ. The Enterprise Gateway Server requires a webMethods Enterprise Gateway license.
Note:
This page is disabled if webMethods API Gateway is installed on Integration Server.
Click Edit Default Alert Options to configure the alerts.
Alert Type
Type of alert Enterprise Gateway Server issues when a request violates an Enterprise Gateway rule. Can be None, Email, or Flow Service.
Send Alert
Frequency with which Enterprise Gateway Server sends an alert when a rule is violated.
This value...
Indicates...
On rule violation
An alert will be sent each time a request violates the rule.
Every number minutes
A limit for the number of alerts sent within a specified time interval. With this option, Enterprise Gateway Server sends an alert the first time a request violates a rule. If the violation occurs again within the specified time interval, the server waits until the end of the time interval before sending another alert. For example, suppose the time interval is 1 minute. At 10:00, a request violates the rule and the server sends an alert. Ten seconds later, another request violates the rule. Ten seconds after that, yet another request violates the rule. The server waits until 10:01 to send a summary of alerts that were generated in the time interval.
Email Addresses
One or more email addresses to which Enterprise Gateway Server sends an alert when a request violates this rule and the alert type is Email. To use this option, the Integration Server acting as the Enterprise Gateway Server must be configured to send emails. Check the settings on the Settings > Resources page under Email Notification. If Enterprise Gateway Server is not configured to send emails, see webMethods Integration Server Administrator’s Guide for instructions on configuring the server to have this ability.
folder.subfolder:service
Name of the flow service to run when a request violates an Enterprise Gateway rule and the alert type is Flow Service. Use the pub.security.enterpriseGateway:alertSpec specification as the signature of the flow service. For more information about the specification, see webMethods Integration Server Built-In Services Reference.
Run As User
Integration Server user ID to run the flow service that Enterprise Gateway Server will invoke when a request violates an Enterprise Gateway rule and the alert type is Flow Service.
