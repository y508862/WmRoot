Edit Keystore Alias
Use the Edit Keystore Alias page to change the properties of a keystore alias used by Integration Server. A keystore contains private keys and certificates for an Integration Server, partner application, or Integration Server component. A keystore alias is a user-specified identifier that can be easily referenced from within Integration Server.
Important:webMethods does not provide tools for the creation and management of keystores; they must be created elsewhere and imported into Integration Server.
Click Return to Keystore to return to the list of keystores that are currently defined on the server.
Keystore Properties
Alias
Displays the user-specified identifier for the keystore. Clicking on the keystore or truststore alias displays its properties.
Description (optional)
Allows the user to specify a description of the alias.
Type
Specifies the file format of the keystore. The default is JKS. You can also use PKCS12. Other keystore types can be made available by loading additional security providers, or by setting the watt.security.keyStoreTypes server configuration property.
Provider
Specifies the provider that is used for the keystore type. The default is the provider shipped with the JVM (Oracle, IBM, or others). If your Hardware Security Module (HSM) device is not supported by the default provider, you can configure a different provider to support keystore file types.
Note:Integration Server supports both PKCS12 and JKS file formats for keystores.
Location
Specifies the location of the keystore file on the server. This can be the full pathname, or a relative path in relation to the Integration Server.
Password/Re-type Password
Specifies the password for the saved keystore file associated with this alias. This password must have been defined at the time the truststore was created, using your keystore utility. Be sure that you have this password available when managing keystore aliases, as you cannot access it through the Integration Server.
HSM Based Keystore
Specifies whether or not the keystore is backed by an HSM-based keystore (a smart card device can be used as well). When the keystore is backed by such a device, the private key does not physically leave the HSM device and certain cryptographic operations must be performed on that device.
Keystore Aliases
These fields are displayed after you click Submit to create the alias, if you have a valid keystore specified.
Alias
Lists each alias found in the keystore.
Password/Re-type Password
Specifies a password for each alias found in the keystore. Most aliases require a password. If Integration Server needs to use this alias for any reason, you must provide its password.
Null
Specifies that the alias is not secured with a password.
Save Changes
Click to submit the password entries for the keystore aliases.
